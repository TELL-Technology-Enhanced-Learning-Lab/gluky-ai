La cartella `docs/` è dedicata alla **documentazione** del progetto e deve contenere materiali utili alla comprensione, all’utilizzo e alla diffusione dei risultati del lavoro svolto.

Ogni sviluppatore o gruppo di lavoro è tenuto a inserire in questa cartella i seguenti elementi:

---

## 📄 Documentazione accademica del progetto

Se il progetto è collegato a una tesi o a un lavoro accademico, inserire in questa cartella:

- ✅ Il **sorgente LaTeX** (cartelle e file `.tex`) se la tesi è scritta in LaTeX oppure il file `.docx` se redatta in Word
- ✅ Il file `.pdf` definitivo della tesi
- ✅ Il file della **presentazione** (PowerPoint `.pptx` o PDF)

Organizza questi file in sottocartelle chiare, ad esempio:
```
docs/
├── latex/
├── word/
├── pdf/
└── presentazione/
```

---

## 🧪 Guida tecnica: `HOWTO.md`

All’interno della cartella `docs/` **deve essere presente** anche un file `HOWTO.md`, con istruzioni tecniche dettagliate per:

1. Come installare e configurare l’ambiente di sviluppo
2. Come eseguire il software
3. Limitazioni note del progetto
4. Errori conosciuti o comportamenti anomali

---

## 📌 Note aggiuntive

- Inserisci eventuali **documenti di progetto**, bozze, report o specifiche in questa cartella.
- Mantieni una **struttura ordinata**: evita file duplicati o non aggiornati.
- Non caricare dati personali o sensibili.

## 📄 Come scrivere il file `HOWTO.md`

Nel file `docs/HOWTO.md` devi fornire indicazioni tecniche chiare per chi, in futuro, dovrà usare o estendere il tuo progetto.  
**Usa questo schema**:

```markdown
# Nome del Progetto

## Autore
Nome Cognome – [@username](https://github.com/username)

### SETUP DEL CODICE
- Librerie usate
- Strumenti richiesti (Python ≥ 3.10, MariaDB, GPU, ecc.)
- Eventuali prerequisiti
- Istruzioni dettagliate per configurare l’ambiente

### ESECUZIONE
- Come lanciare il programma
- Comandi di esempio
- Output attesi

### LIMITAZIONI
- Requisiti minimi
- Funzionalità non testate su altri sistemi
- Dipendenze da componenti specifici

### ERRORI NOTI
- Bug ricorrenti
- Soluzioni temporanee
```
